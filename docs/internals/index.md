## 关键设计原则

在开始之前，先了解这套脚手架背后的四条核心约定，它们贯穿所有阶段的设计决策：

1. **与业务完全解耦** — 脚手架本身不包含任何业务逻辑，`packages/` 下只放示例包，真实业务包在 fork/clone 后自行添加。
2. **按模块拆分** — 通过 `exports` 子路径支持按需引入，每个包只暴露它该暴露的东西。
3. **完整 TypeScript 类型** — 所有公开 API 必须有类型声明，`declaration` + `declarationMap` 同步输出，IDE 跳转定义能直达源码。
4. **复用优先** — 新组件库直接 fork/clone 此模板，零额外配置成本，开箱即用。
